# CHDSharpTest

**xUnit unit and corpus test suite for CHDSharp ‚Äî tests header parsing, CRC checksums, and full decompression against a deterministic 29-file CHD corpus (V1‚ÄìV5, all codecs).**

---

## Test Structure

| Class | Type | Description |
|-------|------|-------------|
| `HeaderAndApiTests` | Unit | Header magic validation, version detection, `CheckHeader`, `IsChdFile`, open/read error paths |
| `ChecksumTests` | Unit | CRC-32 and CRC-16 test vectors |
| `CorpusTests` | Corpus | Runs `CheckFile` and reads metadata from all 29 corpus CHD files in `TestData/`, validates expected version, handles expected-failure entries |

### Test Corpus

The `TestData/` directory contains 29 deterministic CHD files spanning every format version and codec combination, generated by `CHDSharpTestGen`:

| Version | Files | Coverage |
|---------|-------|----------|
| V1 | 1 | zlib (hdcomp legacy) |
| V2 | 1 | zlib (synthesized V1 ‚Üí V2) |
| V3 | 4 | zlib, CD, A/V laserdisc, parent chain |
| V4 | 5 | zlib, uncompressed, CD, A/V laserdisc (patched V3 ‚Üí V4), parent chain |
| V5 | 18 | zlib, lzma, huff, flac, zstd, multi-codec, uncompressed, tiny (expected failure), odd-size, parent, compressed-map child, uncompressed-map child, unaligned-hunk child, CD cdzl/cdlz/cdfl/cdzs/default, laserdisc avhuff |

A `manifest.json` index drives the tests, describing expected version, parent relationships, and expected-pass/fail status for each file.

---

## Running

```bash
# Run all tests
dotnet test

# Run a specific test class
dotnet test --filter "FullyQualifiedName~HeaderAndApiTests"

# Run with verbose output
dotnet test --logger "console;verbosity=detailed"
```
